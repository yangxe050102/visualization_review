## 一、比较与排序类图表 
1. 柱状图（Bar Chart） 
 
① 基础信息 
最经典的类别对比图表，通过柱子的长度映射数值大小，用于不同分类间的数值比较与排序，是商业分析中使用最广泛的图表之一。 
 
② 构成与视觉通道 
- 核心构成：分类轴（横轴/纵轴）、数值轴、矩形柱子、数据标签与图例 
- 视觉通道：核心为长度（柱子高度/宽度），辅助通道为颜色（区分分组/类别）、位置（分类排序） 
 
③ 适用场景 
- 离散分类的数值大小对比、排名展示 
- 多组分类数据的横向/纵向对比 
- 适合类别数量适中（≤15类）的场景，如各部门业绩、各产品销量对比 
 
④ 注意事项 
- 数值轴必须从0开始，截断纵轴会严重误导数值对比 
- 建议按数值大小排序，提升可读性；水平柱状图（条形图）更适合长类别名称 
- 不适合展示占比、连续数据分布 
 
⑤ 优缺点 
- 优点：认知门槛极低，数值对比精度高，制作简单，大众接受度高 
- 缺点：类别过多时会拥挤，信息密度有限，无法展示层级与占比关系 
 
⑥ 常见变体 
水平条形图、分组柱状图、堆叠柱状图、双向柱状图、百分比堆叠柱状图 
#%% 
import matplotlib.pyplot as plt

categories = ['A', 'B', 'C', 'D']
values = [20, 35, 30, 25]

plt.bar(categories, values)
plt.show()

2. 环柱状图（Radial Bar Chart / 环形柱状图） 
 
① 基础信息 
柱状图的极坐标变体，也叫径向柱状图，将分类沿圆周分布，柱子沿径向延伸，用径向长度映射数值，兼具排序对比与视觉表现力。 
 
② 构成与视觉通道 
- 核心构成：圆周分类轴、径向数值轴、环形排布的柱子 
- 视觉通道：核心为径向长度，辅助通道为角度（分类区分）、颜色（分组） 
 
③ 适用场景 
- 类别数量较多的排名展示，如TOP20城市销量排行 
- 强调视觉冲击力的汇报、大屏看板 
- 周期性分类数据（如月度、季度）的对比 
 
④ 注意事项 
- 外圈柱子视觉权重天然更高，会放大高数值的视觉感知，存在误导风险 
- 不适合精确数值读取，仅适合相对大小对比 
- 类别过少（≤5类）使用会浪费空间、形式大于内容 
 
⑤ 优缺点 
- 优点：节省横向空间，视觉表现力强，适合多类别排序展示 
- 缺点：数值精度差，外圈存在视觉放大效应，解读成本高于普通柱状图 
 
⑥ 常见变体 
分组环柱状图、堆叠环柱状图、螺旋柱状图、单指标进度环 
#%% 
import matplotlib.pyplot as plt
import numpy as np

values = [20, 35, 30, 25, 40]

plt.subplot(projection='polar')
plt.bar(angles, values)
plt.xticks(angles, categories)
plt.show()

3. 子弹图（Bullet Graph） 
 
① 基础信息 
由可视化专家Stephen Few设计，是替代仪表盘的紧凑型指标图表，可在极小空间内同时展示实际值、目标值与绩效区间，核心用于KPI对标。 
 
② 构成与视觉通道 
- 核心构成：背景绩效区间（差/良/优分段）、实际值主条、目标值参考线 
- 视觉通道：核心为长度（实际值、区间范围），辅助通道为位置（目标线）、颜色明度（绩效等级） 
 
③ 适用场景 
- 仪表盘、看板中的多指标绩效监控 
- 实际值与目标值的对标分析，如销售完成率、预算执行率 
- 空间有限但需承载多维度指标信息的场景 
 
④ 注意事项 
- 绩效区间划分需符合业务逻辑，通常分为3-5段 
- 目标线与实际值条需视觉区分明确 
- 不适合多类别排序、占比分析 
 
⑤ 优缺点 
- 优点：信息密度极高，紧凑高效，单图承载实际值、目标值、绩效等级三重信息 
- 缺点：认知门槛略高，不适合非专业受众，仅适合单指标对标，不支持多类别对比 
 
⑥ 常见变体 
垂直子弹图、多指标子弹图矩阵、彩色子弹图、反向子弹图 
#%% 
import matplotlib.pyplot as plt

# 绩效区间、实际值、目标值
ranges = [0, 40, 70, 100]
actual = 65
target = 75

fig, ax = plt.subplots(figsize=(6, 1.5))
# 背景绩效分段
ax.barh(0, ranges[1], height=0.5, color='#eee')
ax.barh(0, ranges[2]-ranges[1], left=ranges[1], height=0.5, color='#ccc')
ax.barh(0, ranges[3]-ranges[2], left=ranges[2], height=0.5, color='#aaa')
# 实际值条与目标线
ax.barh(0, actual, height=0.2, color='#333')
ax.axvline(target, color='red', linewidth=2)

ax.set_yticks([])
plt.show()

4. 哑铃图（Dumbbell Chart） 
 
① 基础信息 
也叫哑铃点图，通过线段连接同一主体的两个数据点，直观展示两点间的差值与变化方向，是替代分组柱状图的简洁对比方案。 
 
② 构成与视觉通道 
- 核心构成：数值轴、两个端点（对应两个对比维度）、连接线段 
- 视觉通道：核心为线段长度（差值大小）、端点位置（数值），辅助通道为颜色（差值方向/分组） 
 
③ 适用场景 
- 同一主体的前后对比，如政策实施前后的指标变化 
- 两组数据的差异对比，如不同群体的收入差距、男女各岗位人数差 
- 排名变化、差距分析类场景 
 
④ 注意事项 
- 主体数量不宜过多（≤20个），否则线条重叠混乱 
- 建议按差值大小排序，提升可读性 
- 仅支持两个对比维度，无法承载多时间点数据 
 
⑤ 优缺点 
- 优点：差值感知直观，比分组柱状图更简洁，能清晰展示变化方向 
- 缺点：仅支持双点对比，主体数量受限，信息维度有限 
 
⑥ 常见变体 
水平哑铃图、垂直哑铃图、分组着色哑铃图、棒棒糖图（单端点变体） 
#%% 
import matplotlib.pyplot as plt

plt.yticks(y_pos, categories)
plt.legend()
plt.show()

5. 雷达图（Radar Chart / 蜘蛛图） 
 
① 基础信息 
也叫蜘蛛图、星图，将多个维度映射到极坐标的放射轴上，用多边形轮廓展示多维度数据的整体形态，核心用于多维度综合对比。 
 
② 构成与视觉通道 
- 核心构成：多个等角度分布的数值轴、连接各维度的折线/多边形 
- 视觉通道：核心为径向位置（数值大小）、多边形形状（整体轮廓），辅助通道为角度（维度区分）、颜色（不同系列） 
 
③ 适用场景 
- 多维度能力评估，如员工能力画像、产品属性评分、企业竞争力模型 
- 多主体的综合指标对比，如不同城市的营商环境、不同方案的多维度打分 
- 优劣势分析、综合画像类场景 
 
④ 注意事项 
- 维度数量建议5-8个，过多会导致拥挤、难以识别 
- 维度的排列顺序会显著影响形状感知，需按逻辑排序 
- 数值轴必须从0开始，否则会严重扭曲轮廓 
- 不适合精确数值对比，仅适合整体形态判断 
 
⑤ 优缺点 
- 优点：能直观展示整体优劣势，多维度综合对比的表现力强 
- 缺点：面积与形状感知误差大，维度顺序影响解读，数值精度差，维度过多可读性骤降 
 
⑥ 常见变体 
填充雷达图、多系列雷达图、极坐标面积图、玫瑰图（南丁格尔图，相关变体） 
#%% 
import matplotlib.pyplot as plt
import numpy as np

labels = ['维度1', '维度2', '维度3', '维度4', '维度5']
values = [80, 60, 75, 90, 70]
angles = np.linspace(0, 2 * np.pi, len(labels), endpoint=False)

# 闭合多边形
values += values[:1]
angles = np.append(angles, angles[0])

plt.subplot(projection='polar')
plt.plot(angles, values)
plt.fill(angles, values, alpha=0.2)
plt.xticks(angles[:-1], labels)
plt.show()

6. 平行坐标图（Parallel Coordinates Plot） 
 
① 基础信息 
高维数据探索的核心图表，将每个维度映射为一条平行的数值轴，用折线连接同一样本在各轴上的点，用于发现多维数据的模式、聚类与异常值。 
 
② 构成与视觉通道 
- 核心构成：多条平行的数值轴、连接单样本的折线 
- 视觉通道：核心为轴上位置（数值）、折线形态（数据模式），辅助通道为颜色（样本分组） 
 
③ 适用场景 
- 高维数据的探索性分析，如用户画像聚类、产品多指标对比 
- 异常值识别、模式发现，如不同批次产品的质量参数分布 
- 机器学习特征分析、多维度样本分组 
 
④ 注意事项 
- 维度顺序会直接影响模式发现，需按相关性或业务逻辑排序 
- 样本过多会出现「过绘制」，可通过透明度、采样、交互筛选解决 
- 认知门槛高，不适合面向大众的汇报 
 
⑤ 优缺点 
- 优点：可同时展示十余个维度的数据模式，能有效发现聚类、趋势与异常值 
- 缺点：维度过多时可读性下降，线条重叠严重，解读需要专业背景 
 
⑥ 常见变体 
聚类着色平行坐标图、交互式平行坐标图、水平平行坐标图、平行类别图（适配分类维度）

7. 词云图（Word Cloud） 
 
① 基础信息 
文本数据的可视化形式，通过文字的字号、颜色映射词汇的频率/权重，用于快速展示文本中的高频核心词汇。 
 
② 构成与视觉通道 
- 核心构成：不同大小的词汇、可选的形状边界 
- 视觉通道：核心为字号（文字面积，映射词频），辅助通道为颜色（分类、情感或视觉装饰） 
 
③ 适用场景 
- 文本关键词概览，如舆情热词、评论关键词、报告核心词 
- 封面、大屏等强调视觉效果的场景 
- 快速传递高频信息，不要求精确对比的场景 
 
④ 注意事项 
- 仅适合展示相对频率，无法承载精确数值对比 
- 需提前清洗停用词、无意义词汇 
- 词汇数量建议50-100个，过多会拥挤难读 
- 不适合严谨的数据分析与决策场景 
 
⑤ 优缺点 
- 优点：直观生动，传播性强，快速抓住核心信息，视觉吸引力强 
- 缺点：信息密度低，面积感知误差大，缺乏逻辑顺序，无法展示精确数值与关联 
 
⑥ 常见变体 
形状词云（图云）、词频条形词云、情感着色词云、交互式词云

二、局部与整体类图表 
 
1. 饼图（Pie Chart） 
 
① 基础信息 
最经典的占比可视化图表，通过扇形的圆心角映射类别占比，完整圆形代表整体（100%），核心用于展示「部分占整体的比例」。 
 
② 构成与视觉通道 
- 核心构成：完整圆形、多个扇形、类别标签与百分比 
- 视觉通道：核心为圆心角与扇形面积，辅助通道为颜色（类别区分） 
 
③ 适用场景 
- 分类数量少（≤5类）的占比分析，如市场份额、收入构成、人员结构 
- 强调「部分与整体」关系的场景，适合大众受众 
- 单组数据的构成展示 
 
④ 注意事项 
- 分类数量不建议超过5-7类，过多会导致扇形过小难以区分 
- 所有类别总和必须为100%，必须标注百分比 
- 不适合多组数据对比，不适合精确数值排序 
- 禁止使用3D效果，会严重扭曲面积感知 
 
⑤ 优缺点 
- 优点：大众认知度极高，「占比」的语义直观，形式简单易懂 
- 缺点：角度与面积感知精度差，不适合精确对比，分类过多失效，多组对比困难 
 
⑥ 常见变体 
环形图、爆炸饼图、3D饼图、多饼图阵列

2. 环形图（Donut Chart） 
 
① 基础信息 
饼图的核心变体，将圆形中心挖空，通过环形的弧长映射占比，弱化面积感知、强化弧长感知，同时中心可承载额外信息。 
 
② 构成与视觉通道 
- 核心构成：环形带、扇形分段、中心信息区 
- 视觉通道：核心为弧长/角度，辅助通道为颜色、中心文本 
 
③ 适用场景 
- 同饼图的所有占比场景，更适配现代风格的看板、仪表盘 
- 中心需展示总数、核心指标的场景，如总销售额+各渠道占比 
- 多组小尺寸占比的并列展示 
 
④ 注意事项 
- 与饼图一致，分类不宜过多，禁止3D效果 
- 弧长感知精度仍弱于长度，不适合精确对比 
- 中心信息不宜过多，避免拥挤 
 
⑤ 优缺点 
- 优点：比饼图更节省空间，中心可承载核心指标，视觉更现代简洁 
- 缺点：本质与饼图一致，占比感知精度有限，不适合精确对比 
 
⑥ 常见变体 
多层环形图、单指标进度环、分组环形图、半环形图

3. 旭日图（Sunburst Chart） 
 
① 基础信息 
也叫多层环形图、太阳图，是饼图的层级扩展，通过多层同心环展示层级数据的占比关系，从内到外对应层级的逐步细化。 
 
② 构成与视觉通道 
- 核心构成：内层父类别环、外层子类别环、层级嵌套结构 
- 视觉通道：核心为角度/弧长（占比），辅助通道为径向层级（结构深度）、颜色（父类区分） 
 
③ 适用场景 
- 具有层级结构的占比分析，如「大区-省份-城市」的销售占比 
- 产品分类、组织架构、渠道层级的构成拆解 
- 同时展示整体占比与细分构成的场景 
 
④ 注意事项 
- 层级不宜过深（建议≤3层），否则外圈类别过小难以阅读 
- 子类别数值总和必须等于对应父类别 
- 不适合精确数值对比，仅适合层级占比概览 
 
⑤ 优缺点 
- 优点：可同时展示层级结构与各层级占比，信息密度高于饼图 
- 缺点：层级越深解读成本越高，外圈面积感知误差大，分类过多可读性差 
 
⑥ 常见变体 
冰柱图（垂直矩形版旭日图）、交互式旭日图、径向树图

4. 圆堆积图（Circle Packing Chart） 
 
① 基础信息 
也叫圆形包装图、气泡树图，通过圆形的嵌套与面积映射层级数据的数值大小，大圆包含小圆，直观展示层级包含关系与规模差异。 
 
② 构成与视觉通道 
- 核心构成：父级大圆、子级小圆、嵌套布局 
- 视觉通道：核心为圆形面积（数值大小），辅助通道为嵌套关系（层级）、颜色（分类） 
 
③ 适用场景 
- 层级数据的规模对比，如企业「部门-小组」的人员规模、「品类-单品」的销量 
- 生态系统、组织架构等强调包含关系的层级数据 
- 视觉导向的层级数据展示 
 
④ 注意事项 
- 圆形面积感知存在天然误差，不适合精确数值对比 
- 层级不宜过深，空间利用率较低，空白区域多 
- 小圆难以标注标签，仅适合展示规模差异 
 
⑤ 优缺点 
- 优点：层级包含关系直观，视觉美观，规模差异的感知强 
- 缺点：空间利用率低，数值精度差，无法展示精确占比，信息密度有限

1. 直方图（Histogram） 
 
① 基础信息 
连续型数据分布分析的基础图表，将数值值域划分为连续区间（分箱），用柱子高度表示每个区间的样本频数，展示数据的分布形态。 
 
② 构成与视觉通道

- 视觉通道：核心为柱子高度（频数），横轴位置对应数值区间，整体形态反映分布特征

- 不适合多组数据对比（易拥挤重叠）

- 缺点：分箱选择具有主观性，多组对比效果差，无法精确展示分位数

3. 箱线图（Box Plot / 箱形图） 
 
① 基础信息 
由统计学家John Tukey提出，通过「最小值、Q1、中位数、Q3、最大值」五数概括展示数据分布，核心用于多组分布对比与异常值识别。 
 
② 构成与视觉通道 
- 核心构成：箱体（Q1-Q3四分位距）、中位数线、须线、异常值点 
- 视觉通道：核心为位置（分位数）、长度（离散程度），异常值点单独标记 
 
③ 适用场景 
- 多组数据的分布、离散程度对比，如不同班级成绩、不同地区收入 
- 异常值识别、数据质量校验 
- 空间有限的多组分布对比，是统计分析的标准工具 
 
④ 注意事项 
- 异常值默认按「1.5倍四分位距」判定，需明确统计规则 
- 适合对比，单组数据的信息密度低于直方图 
- 无法展示分布的细节形态（如双峰、偏态细节） 
 
⑤ 优缺点 
- 优点：信息凝练，占用空间极小，多组对比效率极高，能清晰识别异常值 
- 缺点：无法展示分布细节形态，认知门槛高于直方图，非专业受众理解成本高 
 
⑥ 常见变体 
水平箱线图、带缺口箱线图、分组箱线图、分位数箱线图

5. 瀑布图（Waterfall Chart） 
 
① 基础信息 
也叫阶梯图，通过浮动的柱子展示各因素对总量的累计影响，直观呈现从初始值到终值的增减过程，是财务拆解的核心图表。 
 
② 构成与视觉通道 
- 核心构成：首尾总量柱、中间增减浮动柱、累计逻辑线 
- 视觉通道：核心为柱子长度（增减量）、位置（累计值），辅助通道为颜色（增减方向） 
 
③ 适用场景 
- 财务拆解，如利润构成、成本变化、营收增减因素分析 
- 总量的因素拆解，如库存变化、用户增长来源拆解 
- 展示「总量如何一步步变化而来」的逻辑场景 
 
④ 注意事项 
- 各因素需具有可加性，总和逻辑必须自洽 
- 必须标注每个柱子的具体数值

- 缺点：因素过多会拥挤，仅适合可加性数据，适用场景有限 
 
⑥ 常见变体 
水平瀑布图、堆叠瀑布图、小计瀑布图、百分比瀑布图

import numpy as np

nodes = ['A', 'B', 'C', 'D', 'E']
edges = [('A','C'), ('B','D'), ('C','E'), ('A','E')]
x = np.arange(len(nodes))

fig, ax = plt.subplots(figsize=(8, 3))
# 绘制节点
ax.scatter(x, np.zeros(len(nodes)), s=100, zorder=5)
for i, name in enumerate(nodes):
    ax.text(x[i], -0.15, name, ha='center')

# 绘制弧形连线
for u, v in edges:
    i, j = nodes.index(u), nodes.index(v)
    mid_x = (x[i] + x[j]) / 2
    radius = abs(x[j] - x[i]) / 2
    theta = np.linspace(0, np.pi, 100)
    ax.plot(mid_x + radius * np.cos(theta), radius * np.sin(theta), color='gray')

ax.set_ylim(-0.5, max(x) + 0.5)

交互式和弦图、分级和弦图、半和弦图

4. 桑基图（Sankey Diagram） 
 
① 基础信息 
通过节点与流动带展示数据的流向、分流与汇聚，流带宽度对应流量大小，核心是展示从左到右的流程分配、转化与路径。 
 
② 构成与视觉通道 
- 核心构成：层级节点、流动带、流向路径 
- 视觉通道：核心为流带宽度（流量大小）、流向路径，辅助通道为颜色（分类/来源） 
 
③ 适用场景 
- 流量分配与路径分析，如能源流向、用户行为路径、资金流向 
- 多环节的转化与分流分析，如渠道-页面-转化的用户路径 
- 流量守恒的流程拆解 
 
④ 注意事项 
- 流量必须守恒（流入=流出），逻辑需自洽 
- 层级与分支不宜过多，否则会混乱 
- 适合展示流向，不适合精确数值对比 
 
⑤ 优缺点 
- 优点：流向与流量分配逻辑直观，能清晰展示多节点的分流与汇聚 
- 缺点：分支过多可读性下降，制作复杂度高，仅适合流量类数据 
 
⑥ 常见变体 
多层桑基图、对比桑基图、冲积图（Alluvial，分类随时间变化）
